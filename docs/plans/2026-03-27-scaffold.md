# Scaffold Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Create the full folder structure, placeholder pages, component subdirectories, and root config files for the Uncommon Sense streetwear e-commerce site.

**Architecture:** Two route groups — `(store)` for the customer-facing storefront and `(admin)` for the management dashboard — each with their own layout shell. Shared code (components, lib, actions, types) lives at the project root. Server Actions handle all mutations; only the Stripe webhook uses a route handler.

**Tech Stack:** Next.js 16.2.1 (App Router), TypeScript, Tailwind v4, Clerk, Cloudinary, Stripe, Prisma + PostgreSQL

**Brand:** Uncommon Sense — dark theme, muted palette with one accent color, clean typography, lots of whitespace. Mobile-first.

> **Note:** In Next.js 16, `params` on pages and layouts is a `Promise`. Always type it as `Promise<{ param: string }>` and `await` it before use.

```ts
import { NextRequest, NextResponse } from "next/server";
import { stripe } from "@/lib/stripe";

export async function POST(req: NextRequest) {
  const body = await req.text();
  const sig = req.headers.get("stripe-signature");

  if (!sig) {
    return NextResponse.json({ error: "Missing signature" }, { status: 400 });
  }

  let event;
  try {
    event = stripe.webhooks.constructEvent(
      body,
      sig,
      process.env.STRIPE_WEBHOOK_SECRET!
    );
  } catch (err) {
    return NextResponse.json({ error: "Invalid signature" }, { status: 400 });
  }

  switch (event.type) {
    case "checkout.session.completed":
      // TODO: create order in DB
      break;
    default:
      break;
  }

  return NextResponse.json({ received: true });
}
```
